## 프로세스 진도관리의 개요

**프로세스 진도관리**(Progress Control)는 생산계획에 따라 작업이 예정된 일정과 수량대로 수행되고 있는지를 관리하는 활동이다. 진도관리는 생산통제의 핵심 기능으로서 납기 준수, 재공품 감소, 생산성 향상 및 고객만족 확보를 목적으로 한다.

### 진도관리의 정의

진도관리란 생산계획과 실제 작업실적을 비교하여 차이를 분석하고 필요한 조치를 수행하는 관리활동이다.

### 진도관리의 목적

* 납기 준수
* 생산계획 달성
* 재공품 감소
* 생산성 향상
* 병목공정 관리
* 고객서비스 향상

### 진도관리의 중요성

생산계획이 아무리 우수하더라도 실행단계에서 진도관리가 이루어지지 않으면 납기지연, 재고증가 및 생산성 저하가 발생한다.

```mermaid
flowchart LR
    A[생산계획] --> B[작업지시]
    B --> C[진도관리]
    C --> D[실적수집]
    D --> E[차이분석]
    E --> F[시정조치]
```

## 생산통제와 진도관리

### 생산통제의 개념

생산통제(Production Control)는 생산활동이 계획대로 수행되도록 관리하는 기능이다.

### 생산통제의 기능

| 기능   | 내용         |
| ---- | ---------- |
| 작업지시 | 생산작업 배정    |
| 진도관리 | 진행상황 파악    |
| 일정조정 | 일정 변경 및 대응 |
| 실적관리 | 생산실적 분석    |
| 시정조치 | 문제 해결      |

### 생산통제 체계

![https://ko.wikipedia.org/wiki/PDCA](https://upload.wikimedia.org/wikipedia/commons/thumb/7/7a/PDCA_Cycle.svg/500px-PDCA_Cycle.svg.png)

진도관리는 PDCA[^pdca] 사이클의 Check 단계에 해당한다.

[^pdca]: PDCA(plan–do–check–act)는 사업 활동에서 생산 및 품질 등을 관리하는 방법이다. Plan(계획)-Do(실행)-Check(평가)-Act(개선)의 4단계를 반복하여 업무를 지속적으로 개선한다. 월터 슈하트(Walter A. Shewhart), 에드워즈 데밍(W. Edwards Deming) 등에 의해 유명해졌다.

## 진도관리 방식

### 집중식 진도관리

중앙관리부서에서 전체 생산진도를 통합 관리하는 방식

**장점**

* 전체 최적화 가능
* 통제력 우수
* 정보 일관성 확보

**단점**

* 현장 대응 지연
* 관리부하 증가

### 분산식 진도관리

각 공정 또는 작업조직이 직접 진도를 관리하는 방식

**장점**

* 신속한 대응
* 현장 자율성 향상

**단점**

* 부서 간 조정 어려움
* 전체 최적화 한계

### 집중식과 분산식 비교

| 구분   | 집중식 | 분산식 |
| ---- | --- | --- |
| 통제력  | 높음  | 낮음  |
| 대응속도 | 느림  | 빠름  |
| 정보통합 | 우수  | 제한적 |
| 자율성  | 낮음  | 높음  |

## 병목관리(TOC)

![https://www.slideserve.com/laken/toc](https://cdn2.slideserve.com/3754876/slide1-n.jpg)

### 개요

**TOC**(Theory of Constraints)는 시스템 성과를 제한하는 제약요인을 관리하는 기법이다.[^4]

### 기본 개념

병목공정이 전체 생산능력 결정한다.

### TOC 5단계

![https://www.6sigma.us/six-sigma-in-focus/theory-of-constraints/](https://www.6sigma.us/wp-content/uploads/2024/06/toc-1024x900.webp)

1. 제약요인 발견
2. 제약요인 활용
3. 비제약 공정 동기화
4. 제약요인 개선
5. 새로운 제약 탐색

### Drum-Buffer-Rope

![https://ensembletechnology.com/method/operations-toc-overview/production/](https://www.slideserve.com/laken/toc)](https://image2.slideserve.com/3754876/drum-buffer-rope-n.jpg)

1. **드럼**(Drum) : 생산의 박자
   - 의미: 공정 중 가장 느린 부분인 제약 자원(병목)의 작업 일정표
   - 역할: 드럼은 북을 쳐서 행진하는 군대의 보조를 맞추듯, 전체 공장의 생산 속도와 기준을 설정
2. **버퍼**(Buffer) : 공정의 안전장치
   - 의미: 제약 자원(병목)이 멈추지 않고 계속 가동될 수 있도록 병목 공정 앞에 쌓아두는 자재 또는 시간의 여유
   - 역할: 앞 공정에 문제가 생겨도 제약 자원이 쉬지 않도록 보장하여 전체 생산라인의 멈춤을 방지
3. **로프**(Rope) : 자재 투입 통제
   - 의미: 제약 자원의 작업 속도에 맞춰 맨 처음 공정에 자재가 투입되는 시기를 조절하는 신호
   - 역할: 묶여있는 밧줄(Rope)처럼 앞 공정의 과잉 생산을 막고, 제약 자원이 소화할 수 있는 만큼만 원자재를 투입하게 하여 재고를 최소화

!!! note "관련 단원"
    TOC는 「공정개선」에서도 심화 학습한다.

[^4]: Eliyahu Goldratt, *The Goal*, 1984.

## 스마트 진도관리

### MES 기반 진도관리

MES는 생산현장의 작업실적을 실시간 수집·관리하는 시스템이다.

#### 주요 기능

* 작업지시
* 생산실적 수집
* 설비상태 모니터링
* 품질정보 관리

### APS 기반 일정관리

**APS**(Advanced Planning & Scheduling)는 생산능력(Capacity), 자재(Material), 설비(Equipment), 인력(Manpower) 및 각종 제약조건(Constraints)을 동시에 고려하여 최적의 생산일정을 수립하는 고도화된 생산계획 및 일정관리 시스템이다.

기존 MRP가 무한능력계획(Infinite Capacity Planning)을 전제로 자재 중심의 계획을 수립하는 반면, APS는 **유한능력계획**(Finite Capacity Planning)을 기반으로 실제 생산현장의 제약을 반영하여 실행 가능한 생산일정을 생성한다.

#### 등장 배경

전통적인 생산계획 방식은 설비능력, 작업순서, 자재수급 등의 현실적인 제약을 충분히 고려하지 못하여 계획과 실행 간 괴리가 발생하는 문제가 있었다. 특히 다품종 소량생산, 고객 맞춤형 생산, 짧아진 납기 요구 환경에서는 보다 정교한 일정계획이 필요하게 되었으며, 이를 해결하기 위한 시스템으로 APS가 등장하였다.

#### 주요 기능

* 유한능력계획(Finite Capacity Scheduling) [^FCS]
* 설비 부하평준화(Leveling)
* 병목공정 관리(Constraint Management)
* 최적 생산순서 결정(Sequencing)
* 납기 대응 시뮬레이션
* What-if 분석 지원
* 실시간 일정 재조정(Rescheduling)

[^FCS]: 유한능력계획(Finite Capacity Scheduling)은 설비, 인력, 자재 등의 실제 가용능력(Capacity)을 초과하지 않는 범위 내에서 생산일정을 수립하는 계획 방식이다. 즉, 생산계획 수립 시 설비의 가동시간, 작업자 수, 공정능력, 작업순서 등의 제약조건을 고려하여 실행 가능한 생산일정(Feasible Schedule)을 생성하는 기법이다.

#### 계획 수립 절차

```mermaid 
flowchart LR
    A[수요정보 수집] --> B[자재 및 생산능력 확인]
    B --> C[제약조건 분석]
    C --> D[최적 일정 생성]
    D --> E[시뮬레이션 및 검증]
    E --> F[생산계획 확정]
    F --> G[실행 및 모니터링]
```

#### 주요 고려요소

| 구분   | 내용               |
| ---- | ---------------- |
| 설비능력 | 가용 설비시간, 설비부하    |
| 자재   | 재고수준, 자재 입고계획    |
| 인력   | 작업자 수, 숙련도       |
| 공정   | 작업순서, 표준시간       |
| 납기   | 고객 요구 납기일        |
| 제약조건 | 병목공정, 셋업시간, 우선순위 |

#### APS와 MRP 비교

| 구분      | MRP      | APS           |
| ------- | -------- | ------------- |
| 계획 방식   | 자재 중심    | 자재 + 능력 중심    |
| 능력 고려   | 무한능력계획   | 유한능력계획        |
| 제약조건 반영 | 제한적      | 적극 반영         |
| 일정 생성   | 계획 중심    | 실행 가능 일정 중심   |
| 시뮬레이션   | 미흡       | What-if 분석 지원 |
| 납기 대응   | 상대적으로 낮음 | 우수            |


#### 기대효과

* 생산능력 활용도 향상
* 병목공정 최소화
* 납기준수율 향상
* 재공품(WIP) 감소
* 리드타임 단축
* 생산계획의 실행력 향상

APS는 **"유한능력계획(Finite Capacity Planning)을 기반으로 설비, 자재, 인력 및 제약조건을 동시에 고려하여 최적의 생산일정을 수립하는 지능형 생산계획 시스템"**이다. 최근에는 APS가 MES, ERP, IoT, AI와 연계되면서 실시간 데이터 기반의 동적 일정계획(Dynamic Scheduling)과 자율형 생산운영 체계의 핵심 플랫폼으로 발전하고 있으며, 스마트팩토리 구현의 핵심 요소로 평가받고 있다.

## 프로세스 진도관리의 종합

프로세스 진도관리는 생산계획을 현장에서 실행하고 통제하는 핵심 관리기능이다. 작업지시, 진척도 측정, 병목관리 및 납기관리를 통해 생산성과 고객만족을 확보하며, 최근에는 MES·APS·AI 기반 스마트 생산통제로 발전하고 있다. 특히 진도관리는 생산계획과 현장운영을 연결하는 핵심 고리로서 공장관리의 실질적 실행력을 결정하는 요소이다.
